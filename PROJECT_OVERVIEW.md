# 「忍了么」项目完整框架说明

## 1. 产品定位

「忍了么」是一款温柔坚定的反冲动 iOS App。它帮助用户在冲动消费、冲动进食、冲动娱乐/拖延之前，把欲望记录下来，并把“忍住”的结果转化成可见资产。

核心价值不是管教用户，而是帮助用户重新获得选择权。

产品主张：

- 不是不配拥有，是先问问自己真的需要吗。
- 忍住不是亏待自己，是把选择权拿回来。
- 破防也算数据，它帮你更了解自己。

第一版目标：

- 快速记录一次冲动。
- 清晰量化忍住价值。
- 让用户看到自己已经拿回了什么。
- 避免羞辱、惩罚和过度游戏化。

第一版不做：

- 真实银行/支付账单同步。
- 复杂食物识别能力。
- 社交排名。
- AI 聊天教练。
- 强制戒断或惩罚系统。

## 2. 核心概念

App 中的“资产”不是用户实际拥有的钱，而是用户忍住之后保存下来的价值。

三类资产独立存在，不互相换算：

- 金钱资产：忍住购买后累计的金额，例如 `¥100`。
- 热量资产：忍住食物后累计的热量，例如 `100 kcal`。
- 时间资产：忍住娱乐/拖延后累计的时间，例如 `60 min` 或 `1 h`。

资产计算规则：

- 只有 `我忍住了` 的记录会计入资产。
- `先放进冷静箱` 暂不计入资产。
- `我还是做了` 只作为观察数据，不扣分。
- 资产由记录实时计算，不单独维护一个余额字段。

## 3. 信息架构

当前 App 使用 4 个底部 Tab。

### 首页

用途：给用户即时反馈，让用户看到自己已经拿回来的东西。

当前内容：

- 今日忍住次数。
- 三类忍耐资产总览。
- 目标进度卡。
- 最近记录列表。
- 右下角「忍一下」快速入口。

首页设计原则：

- 大数字展示资产。
- 文案温柔但有行动感。
- 不制造焦虑，不暗示用户失败。
- 三类单位分别展示，不合并成一个总分。

### 记录

用途：完成一次冲动记录。

当前流程分 3 步：

1. 选择类型：金钱、食物、时间。
2. 填写价值：名称、金额/热量/时长、原因、备注、目标。
3. 做决定：我忍住了、先放进冷静箱、我还是做了。

记录页设计原则：

- 输入项尽量少。
- 原因使用预设选项，降低记录成本。
- 食物类文案必须避免羞辱和减肥焦虑。
- “我还是做了”不是失败，只是一次观察。

### 目标

用途：把忍住的价值投向具体目标。

当前内容：

- 默认种子目标：
  - 新相机基金。
  - 本周少喝 3 杯奶茶。
  - 本周拿回 10 小时。
- 新增目标。
- 目标进度展示。
- 删除目标。

目标计算规则：

- 如果记录关联了某个目标，目标优先展示关联记录的进度。
- 如果没有关联记录，当前首页和目标卡会展示同类型总资产作为进度参考。

### 我的

用途：轻量复盘、查看统计、处理冷静箱。

当前内容：

- 温柔复盘文案。
- 本周/本月基础统计。
- 三类本周资产统计。
- 成就卡片。
- 冷静箱列表。

复盘原则：

- 不用“失败”“堕落”“管不住自己”等词。
- 帮用户发现模式，而不是评价人格。
- 把破防记录视为理解自己的数据。

## 4. 数据模型

当前使用 SwiftData 做本地存储。

### ResistRecord

表示一次冲动记录。

字段：

- `id`: 唯一 ID。
- `typeRaw`: 类型原始值，money / food / time。
- `title`: 物品、食物或行为名称。
- `value`: 数值。
- `unitRaw`: 单位原始值，cny / kcal / minute。
- `statusRaw`: 状态原始值，resisted / pending / gaveIn。
- `reason`: 触发原因。
- `createdAt`: 创建时间。
- `resolvedAt`: 做出最终决定的时间。
- `cooldownUntil`: 冷静箱到期时间。
- `enteredCooldown`: 是否曾进入冷静箱，用于成就判断。
- `note`: 用户备注。
- `goalId`: 关联目标 ID。

状态含义：

- `resisted`: 用户忍住了，计入资产。
- `pending`: 放进冷静箱，暂不计入资产。
- `gaveIn`: 用户还是做了，只记录观察数据。

### Goal

表示一个目标。

字段：

- `id`: 唯一 ID。
- `title`: 目标标题。
- `typeRaw`: 目标类型，money / food / time。
- `targetValue`: 目标值。
- `deadline`: 截止时间，当前 UI 暂未完整使用。
- `icon`: SF Symbol 图标名。
- `createdAt`: 创建时间。

## 5. 类型和单位

### ResistType

三种类型：

- `money`: 金钱类。
- `food`: 食物类。
- `time`: 时间类。

每种类型定义：

- 展示标题。
- 资产标题。
- 输入字段标题。
- 单位。
- SF Symbol 图标。
- 主题色。
- 预设原因。
- 温柔提示文案。
- 默认冷静时间。

默认冷静时间：

- 金钱类：24 小时。
- 食物类：10 分钟。
- 时间类：15 分钟。

### ValueUnit

当前单位：

- `cny`: 人民币。
- `kcal`: 千卡。
- `minute`: 分钟。

### ResistStatus

当前状态：

- `resisted`: 忍住了。
- `pending`: 冷静箱。
- `gaveIn`: 观察中。

## 6. 关键流程

### 新增一条“我忍住了”

1. 用户进入记录页。
2. 选择类型。
3. 填写名称和数值。
4. 选择原因，可选备注和目标。
5. 点击「我忍住了」。
6. 创建 `ResistRecord(status: .resisted)`。
7. 首页资产实时重新计算。

预期结果：

- 金钱类增加金钱资产。
- 食物类增加热量资产。
- 时间类增加时间资产。

### 新增一条“先放进冷静箱”

1. 用户填写记录。
2. 点击「先放进冷静箱」。
3. 创建 `ResistRecord(status: .pending)`。
4. 设置 `cooldownUntil`。
5. 设置 `enteredCooldown = true`。
6. 请求通知权限。
7. 到期后发送本地通知。

预期结果：

- 暂不计入资产。
- 在「我的」页冷静箱中出现。
- 用户可稍后改为「我忍住了」或「我还是做了」。

### 新增一条“我还是做了”

1. 用户填写记录。
2. 点击「我还是做了」。
3. 创建 `ResistRecord(status: .gaveIn)`。

预期结果：

- 不计入资产。
- 不扣分。
- 可用于后续复盘。

## 7. 统计规则

统计逻辑集中在 `StatsCalculator`。

当前支持：

- 全量资产统计。
- 某目标当前值统计。
- 某类型总资产统计。
- 今日忍住次数。
- 本周忍住记录。
- 本月忍住记录。
- 最常见触发原因。

统计原则：

- 所有资产统计只计算 `status == .resisted`。
- 周/月统计使用系统 Calendar。
- 时间资产内部以分钟存储，展示时可转换为小时。

## 9. 视觉和语气规范

视觉方向：

- 干净、轻盈、温柔但不幼稚。
- 背景使用浅灰白。
- 卡片使用白色和轻阴影。
- 主强调色为柔和紫色。
- 金钱类偏绿色。
- 食物类偏珊瑚红。
- 时间类偏蓝紫色。

语气原则：

- 不羞辱。
- 不制造焦虑。
- 不惩罚用户。
- 多使用“选择权”“拿回来”“先等等”“观察”等词。

食物热量策略：

- 当前先采用轻量化方案：本地启动食物库 + 包装营养标签手动录入。
- 用户选择食物后填写份量，App 自动计算本次热量。
- 不提供联网搜索入口，避免产品变重。
- 找不到食物时，优先使用模板默认热量、换用相近条目，或后续补充到本地数据库。

避免词：

- 失败。
- 堕落。
- 管不住。
- 没自控力。
- 你胖了。
- 浪费人生。

## 12. 验收标准

基础功能验收：

- 新增一条 `¥100` 金钱记录并选择「我忍住了」，首页金钱资产增加 `¥100`。
- 新增一条 `500 kcal` 食物记录并选择「我还是做了」，首页热量资产不增加。
- 新增一条 `60 min` 时间记录并选择「先放进冷静箱」，首页时间资产不增加，冷静箱出现该记录。
- 在冷静箱中把时间记录改为「我忍住了」，首页时间资产增加 `1 h`。
- 三类资产单位不混淆。
- 关闭并重新打开 App 后，本地记录仍然存在。

体验验收：

- 食物类文案不出现羞辱或体型焦虑表达。
- `gaveIn` 状态不出现惩罚性反馈。
- 首页能在 5 秒内让用户理解自己“拿回了什么”。

## 13. 当前环境说明

当前仓库已经包含 Xcode 工程文件，但本机命令行环境的 active developer directory 是 CommandLineTools，不是完整 Xcode。

因此：

- `plutil -lint RenLeMe.xcodeproj/project.pbxproj` 可验证工程文件格式。
- `xcodebuild` 当前不可用。
- 推荐用完整 Xcode 打开 `RenLeMe.xcodeproj` 后选择 iPhone 模拟器运行。
