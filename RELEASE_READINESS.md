# 忍了么上线准备清单

更新时间：2026-05-19

这份文档用于把当前 Demo 推向 App Store/TestFlight。原则是先完成可上线 MVP，同时保留后续功能和页面的扩展空间。

## 当前上线策略

第一版保留轻量边界：

- 不做账号系统。
- 不做云同步。
- 不做联网食物搜索。
- 不做 AI 聊天教练。
- 不做付费订阅。
- 不做社交排名。

后续可扩展：

- 用户自建食物库。
- 冷静箱提醒设置页。
- 周/月趋势图和更完整复盘。
- 本地数据导出。
- AI 复盘摘要。

## 已处理

- App 图标已存在。
- 自定义启动页已存在。
- 首次新手引导已存在。
- 相机、相册、通知权限文案已写入 Xcode project。
- 本地食物库不联网。
- Demo 假记录只在 Debug 构建自动插入，Release/TestFlight 默认不插入。
- SwiftData 本地保存记录、目标、食物库。
- App Store 文案草稿已补充：`APP_STORE_LISTING_DRAFT.md`。

## 已验证

- Debug 模拟器构建并启动通过。
- Release 模拟器编译通过。
- Release 模拟器首次启动后没有自动插入 Demo 假记录。
- Release 首屏资产为 `¥0 / 0 kcal / 0 min`，默认目标仍保留。

## P0：提交 TestFlight 前必须完成

- 使用 Release 配置构建，确认首次安装不会出现 Demo 假记录。
- 首次启动流程：启动页 -> 新手引导 -> 空/初始首页。
- 新增金钱记录，选择“我忍住了”，首页金额资产增加。
- 新增食物记录，使用模板默认 kcal，首页热量资产增加。
- 新增时间记录，填写分钟，首页时间资产增加。
- 新增冷静箱记录，确认通知权限请求、我的页冷静箱出现。
- 冷静箱处理为“我忍住了”和“我还是做了”都能正常更新。
- 历史页筛选、详情、编辑、删除都正常。
- 新建/编辑/删除目标不会导致崩溃。
- 相册选择自选道具图片正常。
- 真机拍照选择自选道具图片正常。
- 键盘可收起，不遮挡第 4 步决策按钮。
- iPhone 小屏和大屏都无明显文字重叠。

## P1：上架材料

- App Store 名称：忍了么。
- 副标题：冲动来临时，先忍一下。
- 类别建议：Lifestyle。
- 关键词：自律、习惯、记录、消费、饮食、时间管理、情绪、目标。
- 隐私政策 URL：待部署。
- 支持 URL：待部署。
- App Store 截图：至少准备 iPhone 主尺寸截图。
- App 描述：突出轻量记录、冷静箱、忍耐资产、小忍反馈。
- 年龄分级：按无敏感内容填写。

## P2：建议但可后置

- 给 `StatsCalculator` 补单元测试。
- 给记录保存流程补最小 UI 测试。
- 隐私政策页面部署到可公开访问 URL。
- 补一页“数据只保存在本机”的 App 内说明。
- 补充 App Store 审核备注，解释相机/相册/通知用途。

## 权限说明

当前权限用途：

- 相机：用于拍摄自选道具图片。
- 相册：用于选择自选道具图片。
- 通知：用于冷静箱到期提醒。

注意：

- App 不读取支付账单。
- App 不读取健康数据。
- App 不上传记录到服务器。
- App 不做广告追踪。

## 不建议第一版加入

- 登录注册。
- 云同步。
- 复杂 AI Agent。
- 商业化订阅。
- 社区分享。
- 排行榜。
- 联网食物搜索。
